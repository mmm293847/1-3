# 3-1 제출용 README


## 0. 프로젝트 개요

```
"구글 폼에 응답이 작성완료되면 이메일로 알림을 주는 업무 자동화"
```
> 💡 이 README가 어떤 과제인지 한두 줄로 소개. "어떤 반복 업무를 자동화했는지" 한 문장 요약.

---

## 1. 프로젝트 1 — 자동화 도구 비교 구현

### 1-1. 구현한 워크플로우 개요

```
Google Form 응답
↓
분기 별 Google Sheets 저장
↓
분기에 따른 응답 작성 완료 이메일 전송
```


### 1-2. 사용한 도구 목록

---
- 도구 1: Make
- 도구 2: Zapler
---

### 1-3. 도구 1 — 워크플로우 구성

```
Make
- Trigger: Google Form - Watch Responses in Google Sheets

- Router
  - Filter 1: 만족 응답(전반적인 만족도>=3)
  - Filter 2: 불만족 응답(전반적인 만족도<3)

- Action 1: Google Sheets - Add a Row
Route1 : 테스트용 만족 시트 → 만족시트
Route2 : 테스트용 불만족 시트 → 불만족시트

- Action 2: Gmail - Send an email
Route1
→ Subject: 구글시트에 만족응답이 작성되었습니다.
→ Content: 만족시트 연결
Route2
→ Subject: 구글시트에 불만족 응답이 작성되었습니다.
→ Content: 불만족시트 연결
```

---

### 1-4. 도구 1 — 화면 캡처

```
[ 워크플로우 구성 화면 캡처 이미지 첨부 ]
[ 실행 결과 화면 캡처 이미지 첨부 ]
[ 조건 분기 각 경로가 1회 이상 실행된 결과 캡처 이미지 첨부 ]
```
> 💡 캡처 이미지에 API Key, 토큰, 비밀번호, 계정 이메일이 보이면 반드시 **\*\*\*** 로 가림 처리할 것.

---

### 1-5. 도구 2 — 워크플로우 구성

```
Zapler
- Trigger: Google Forms-New Form Response

Path1: Always Run(긍정응답)
→ Filter: 전반적인 만족도 >2
Path2: Always Run(부정응답)
→ Filter: 전반적인 만족도 <3

- Action 1: Google Sheets - Create Speradsheet Row
긍정응답 Path: 테스트용 만족 시트 → 만족시트
부정응답 Path: 테스트용 불만족 시트 → 불만족시트

- Action 2: Gmail - Send Email
긍정응답 Path
→ Subject: [Zapler]만족도 조사에 대한 긍정 응답이 도착하였습니다.
→ Body: 만족시트 연결
부정응답 Path
→ Subject: [Zapler]만족도 조사에 대한 부정 응답이 도착하였습니다.
→ Body: 불만족시트 연결

```
> 💡 도구 1과 동일한 워크플로우 구조로 만들었는지 확인. Trigger·분기·Action 구성이 대응되어야 함.

### 1-7. 비교 분석 보고서

#### 비교 표 (5개 이상 항목 필수)

| 비교 항목 | Make | Zapler |
|---|---|---|
| UI/UX | [ 아이콘 형식 ] | [ 트리형식의 고정형 구조, 직관적임 ] |
| 설정 난이도 | [ 보통 ] | [ 쉬움 ] |
| 연동 서비스 범위 | [ 넓음 ] | [ 한정적 ] |
| 무료 플랜 범위 | [ 3000개 이상의 연결 가능 앱, 시나리오 작동 시간 제한, 파일사이즈 5MB 제한, 크레딧 1000개 제한 ] | [ 기간제 유료플랜 사용, 매달 100개의 프로젝트 생성가능, 접근 가능 파일 제한 ] |
| 실행 로그 확인 방식 | [ 테스트 실행 후, 아이콘 옆에 뜨는 말풍선으로 input과 output값 확인 가능 ] | [ Test run에서 데이터값 설정 후 실행, 테스트별 로그 확인 가능 ] |

---

#### 도구 1 장단점

```
Make
- 장점: 높은 자유도, 다양한 설정 가능 서비스
- 단점: 시각화에 중심을 두어, 오류가 생겼을 때 문제와 이유를 알기 어려움
```

---

#### 도구 2 장단점

```
Zapler
- 장점: 고정형 트리구조로 흐름과 오류 파악 쉬움
- 단점: 한정적인 서비스 수와 정렬되지 않은 데이터값들
```

---

#### 도구 선택 의견 — 어떤 상황에 어떤 도구가 적합한가?

```
Make는 실제 업무에서 사용가능한 도구를 찾고있다면 적합할 것이다.
다양한 서비스와 액션기능의 범위가 상당히 넓기 때문에, 어느 정도 숙지를 한 상태로 들어간다면 효과적인 결과를 도출할 것으로 기대된다.
Kapler는 짧은 기간동안 진행하는 간단한 반복작업에 적합하다.
```
